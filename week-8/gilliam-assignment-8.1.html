Describe what “Cache-Control” is and how it is used.

"Cache-control is an HTTP header used to specify browser caching policies in both client requests and server responses. Policies include how a resource is cached, where it’s cached and its maximum age before expiring. (Imperva, 2018)"

List and describe the available options for managing security in a REST API.

Authentication
A process where a user must provide proof that they have permission to access the system, either with local credentials or through a third party system.

Basic Auth
Uses a simple Base64 encrypted username/password string in the header, under the Authorization key.

Digest
Uses an MD5 encrypted hash of the username/password string in the header, under the Authorization key.

OAuth 1.0
It is a third party system that allows access by the use of tokens instead of the users credentials.

OAuth 2.0
Is simular to OAuth 1.0, but separation of resource delivery and authorization, and allows for multiple types of implementations. This means that it can be utalized by more types of applications, and not just web browsers.
