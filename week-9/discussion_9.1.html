Explain the difference between authentication and authorization.

Authentication is the process where the user needs to prove they are who they say they are.

Authorization is the process where the user just proves they have permission.

Select two HTTP status codes and describe how they are used and why they are important. You CANNOT use 404, 500, and 200.

Status Code: 100

Importance: low - medium

This is the status code that is given when utalizing the LBYL request. This signifies that the system acnologies that you are wanting to make a request, the system can take the request, and you may make the request.

The request headers need to be set to "100-continue".

Status Code: 301

Importance: medium

This is the status code that is given when the resource requested has been permanently moved to a different location. This prevents a URL from breaking when the underling structure of the API changes.

The request headers should put the canonical URL in Location.

Entity-body should send a hypermedia document that links to the new location.

Status Codes came from RESTful Web APIs by Richardson. (Richardson, 2013)
